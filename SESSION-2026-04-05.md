# Pi Backup Manager ‚Äî Session Notes 2026-04-05

## Changes Made

### 1. Reset & Cleanup ‚Äî sudo for root-owned files
**Problem:** Sentinel (`/mnt/backups/.image_initialised`) and lock file (`/tmp/weekly_image.lock`) are created by root (via the backup script running under cron/sudo). Flask app running as `admin` couldn't delete them ‚Äî got EPERM/EACCES.

**Fix:** All four cleanup targets now use `sudo rm -f` via `sh()`, consistent with how `compact_tmp` already worked. Added `import shlex` and used `shlex.quote()` for safe path handling.

### 2. Reset & Cleanup ‚Äî Select All checkbox
Added a "Select all" checkbox at the top of the list with a divider separator:
- Checking it checks all four items; unchecking clears all
- Individual checkboxes keep "Select all" in sync (checked only when all four ticked)
- After successful cleanup run, all five checkboxes clear

### 3. Reset & Cleanup ‚Äî Image label uses wildcard
The image file label now shows `{mountPoint}/*.img` instead of the specific configured filename, to make clear it applies to whatever image is configured.

> **Bug (fixed in session 2):** Only the label was changed to `*.img` ‚Äî the actual backend delete still used the specific `imageName` from config. If config didn't match the real filename (e.g. config had `pi_backup.img` but file was `pi4_backup.img`), `rm -f` would silently succeed on a nonexistent path and report "deleted" incorrectly.

### 4. Reset & Cleanup ‚Äî Layout fixes
- `align-items:flex-start` + `margin-top:2px` on checkboxes ‚Äî pins checkbox top-left when label text wraps
- `word-break:break-all` on `<code>` paths (`.image_initialised`, `/tmp/pbm_compact_mnt`, image path)
- Button + status use `flex-wrap:wrap` so status drops below button on narrow screens

`.tabbar-inner` now has `overflow-x:auto` with hidden scrollbar (`scrollbar-width:none` / `::-webkit-scrollbar{display:none}`). Tabs never overflow off-screen.

### 7. Dashboard tab
New `üìä Dashboard` tab (first in the bar). Loads on page open and on tab click, with a manual Refresh button.

**Backend:** `GET /api/dashboard` ‚Äî aggregates in one call:
- `last` ‚Äî from `api_backup_last()` (result, timestamps, elapsed, log snippet)
- `image` ‚Äî logical MB, sparse MB (via `du`), compact_recommended flag
- `mount` ‚Äî `findmnt` for source + fstype
- `cron` ‚Äî parses `crontab -l` for the script entry, returns expr
- `sentinel` / `lock` ‚Äî boolean existence checks
- `running` ‚Äî from `_backup_status`

**Frontend cards:**
- **Last Backup** ‚Äî result badge, started/finished/duration/mode, scrollable log snippet
- **Mount** ‚Äî mounted/not-mounted badge, source device, fstype
- **Image** ‚Äî present/not-found badge, logical size, on-disk (sparse) size, sentinel status, compaction warning if >500 MB reclaimable
- **Schedule** ‚Äî cron installed badge, expression, computed next run date
- **State pills** ‚Äî backup running/idle ¬∑ lock present/absent ¬∑ sentinel present/absent

## Bug Fixes (Session 2 ‚Äî 2026-04-05)

### 8. Reset & Cleanup ‚Äî Image delete actually wildcards now
**Problem:** Backend `api_cleanup` still used `cfg["imageName"]` for the delete path. The label showed `*.img` but the delete was an exact path. If config didn't match the real filename, `rm -f` exited 0 silently and the API reported "deleted" even though nothing was removed.

**Fix:** Backend now globs `Path(mount_point).glob("*.img")` and deletes all matches. Response now reports `"not found"` (no `.img` files present) or `"deleted N file(s)"` instead of the misleading blanket "deleted".

## Bug Fixes (Session 3 ‚Äî 2026-04-05)

### 9. Dashboard ‚Äî Mount always showed "not mounted"
**Bug:** `api_dashboard` unpacked `sh()` return as `_, out, rc` ‚Äî `out` was stderr (empty), stdout was discarded.
**Fix:** Swapped to `out, _, rc`.

### 10. Dashboard ‚Äî Cron always showed "not set"
**Bug:** Same `_, out, rc` swap on `crontab -l` call ‚Äî stdout discarded, loop iterated over empty string.
**Fix:** Swapped to `out, _, rc`.

### 11. Dashboard ‚Äî Image sparse size always wrong
**Bug:** Same `_, out, _` swap on `du` call ‚Äî `out` was stderr, sparse size always fell back to logical size.
**Fix:** Swapped to `out, _, _`.

### 12. Dashboard ‚Äî Last backup showed "no data" during running cron backup
**Bug:** `_backup_status["running"]` is only set when backup is launched via the web UI, not by cron. Lock file is the reliable signal for a cron-launched backup.
**Fix:** `running` field in `/api/dashboard` response is now `_backup_status["running"] or lock_exists`.

### 13. Dashboard ‚Äî Started time + ETA + live "Running for" counter
When a backup is running, the Last Backup card now shows:
- **Started** ‚Äî parsed timestamp from log (`started_ts` now included in `api_backup_last` response)
- **ETA** ‚Äî label replaces "Finished"; shows `started + prev_elapsed` with "(est.)" suffix. `prev_elapsed` is found by scanning backwards past the current start marker for the previous "All done. Total elapsed: Xh Xm Xs" line.
- **Running for** ‚Äî label replaces "Duration"; live ticking counter (1s `setInterval`, cleared on next `loadDashboard` call)
- Helper functions added: `parseElapsedSecs(s)`, `fmtElapsed(s)`, module-level `_dbTimer`
