# Pi Backup Manager — Session Notes 2026-04-12

## Context

Scheduled backup failed overnight. ntfy reported `Backup FAILED (0h 05m 27s) exit 1`.
Log diagnosis revealed the root cause and several related bugs.

---

## Root Cause — Backup Failure

```
rsync: write failed on "/tmp/img-backup-mnt/home/admin/runtipi/backups/migrated/immich/immich-migrated-...tar.gz": No space left on device (28)
```

The auto-resize logic was fundamentally broken — it resized based on image *free space*
vs a threshold, but never checked whether the *total image capacity* was large enough to
hold the source data. The image grew by 1310 MB pre-backup but still overflowed because
a large Immich tar exceeded the remaining headroom.

---

## Fixes Applied (all in `server.py` script template)

### Fix 1 — `auto_resize_image` logic rewrite (Critical)

**Old behaviour:** compared `image_free_mb` against `headroom / 2` as threshold.
`src_used_mb` was captured but never used in the resize decision (dead code).

**New behaviour:** compares `image_total_mb` (total partition capacity) against
`src_used_mb + headroom_mb` (what is actually needed). Grows by the gap + 10% buffer.

```bash
# OLD (wrong)
threshold_mb=$(( extra_mb / 2 ))
if [ "$free_mb" -ge "$threshold_mb" ]; then ... fi
grow_mb=$(( extra_mb - free_mb + (extra_mb / 10) ))

# NEW (correct)
needed_mb=$(( src_used_mb + extra_mb ))
if [ "$image_total_mb" -ge "$needed_mb" ]; then ... fi
grow_mb=$(( needed_mb - image_total_mb + (extra_mb / 10) ))
```

### Fix 2 — `_pbm_ds_err` permission denied

`smart_start` redirected docker stderr to `/tmp/_pbm_ds_err`. After the first cron run
(as root), that file is root-owned and all subsequent calls fail with `Permission denied`,
silencing all container restart error messages.

**Fix:** Use `mktemp /tmp/_pbm_ds_err.XXXXXX` inside `smart_start` so each call gets
its own temp file owned by the current user. Cleaned up with `rm -f` on all exit paths.

### Fix 3 — Stale loop devices

`auto_resize_image` attached the image via `losetup -fP --show` without first checking
whether a stale loop device from a crashed prior run was still attached to the same file.

**Fix:** Added `sudo losetup -j "$img" | cut -d: -f1 | xargs -r sudo losetup -d` before
each `losetup -fP --show` call (both size-check and resize passes).

### Fix 4 — `smart_start` on already-running containers

`restart_containers` calls `runtipi-cli start` which brings everything up, then
immediately calls `smart_start` on the same containers. If they're already running,
`docker start` errors → compose up -d fallback → noisy log spam.

**Fix:** Added early return at top of `smart_start`:
```bash
[ "$(docker inspect -f '{{.State.Status}}' "$container" 2>/dev/null)" = "running" ] && return 0
```

### Fix 5 — Runtipi API URL (Traefik blocking `/api/` paths)

The generated script set `TIPI_URL` to the user-configured external IP (e.g.
`http://192.168.20.111`). All API calls via this URL returned 404 because Traefik's
`api@internal` router intercepts `PathPrefix('/api')` before the request reaches the
backend.

**Investigation:** Confirmed by testing `http://192.168.20.111/api/auth/login` (404 via
Traefik) vs `http://172.18.0.4:3000/api/auth/login` (201 + cookie, direct container IP).

**Fix:** Removed static `TIPI_URL` variable. Added `_tipi_base_url()` helper that
resolves the container IP at runtime via `docker inspect`:

```bash
_tipi_base_url() {
  local ip
  ip=$(docker inspect -f '{{range.NetworkSettings.Networks}}{{.IPAddress}}{{end}}' runtipi 2>/dev/null | awk '{print $1}')
  [ -n "$ip" ] && echo "http://${ip}:3000" || echo ""
}
```

All API calls in `runtipi_login` and `runtipi_start_app_by_compose` use `$(_tipi_base_url)`.

**API endpoints confirmed working (Runtipi v4.8.2):**
- `POST /api/auth/login` → `{"success":true}` + `Set-Cookie: runtipi-sid=...`
- `GET /api/apps/installed` → installed apps, urn format: `{appName}:{storeSlug}`
- `POST /api/app-lifecycle/{urn}/start` → `{"requestId":"..."}`

### Fix 6 — Default headroom raised

`IMAGE_HEADROOM_MB` default raised from **2000 → 5000 MB** in 4 places:
- `server.py` Python backend (`cfg.get("imageHeadroom", 5000)`)
- HTML input default value
- JS `collectConfig()` fallback
- JS script template fallback

UI hint updated to explain the new resize logic and mention large app backups (Immich).
